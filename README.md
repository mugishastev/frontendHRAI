# HRAI - AI-Powered Talent Platform (Frontend Interface) 🚀🎨

Welcome to the **HRAI Frontend**, a premium, high-performance recruitment dashboard and applicant portal built with **Next.js 15**. It delivers a "Wow" first impression for both hiring managers and job seekers.

---

## 🌟 Premium Features

### 🏢 Recruiter Control Center
*   **4-Column Grid View**: A dense, scan-efficient layout for managing active hiring roles.
*   **AI Insight Matrix**: Interactive scorecards showing Gemini-generated strengths and skill gaps for every applicant.
*   **Batch Operations**: Trigger AI screening across hundreds of applicants with a single click and real-time loading feedback.
*   **Global Insights**: Dynamic charts visualizing application trends and recruitment pipeline health.

### 👤 Modern Applicant Experience
*   **Professional Careers Portal**: Jobs displayed in beautiful Table or Grid views with instant filtering.
*   **AI-Enhanced Search**: Find roles based on skills, department, and location.
*   **Unified Dashboard**: Applicants can track their hiring status, view AI-ranked results, and manage multiple applications.

---

## 🛠️ Technical Stack

*   **Framework**: Next.js 15 (App Router) with React Server Components.
*   **State & API**: Redux Toolkit (RTK) & RTK Query for lightning-fast data synchronization.
*   **Design System**: Tailwind CSS with custom **"Glassmorphism"** aesthetics and 32px premium rounding.
*   **Iconography**: Lucide React for consistent, high-fidelity visual elements.
*   **Deployment**: Optimized for Vercel with server-side rendering for peak SEO.

---

## 📂 Navigation Map

*   `/careers`: Public-facing job board with Table/Grid toggle.
*   `/dashboard`: Main recruiter workspace for active hiring overview.
*   `/manage-jobs`: Dedicated grid view for job role administration.
*   `/batch-ops`: AI Screening control center.
*   `/applicants/dashboard`: Personal portal for candidate status tracking.

---

## 🚀 Local Development

1. **Install Dependencies**: `npm install`
2. **Environment Setup**: Create `.env.local` and set `NEXT_PUBLIC_API_URL` to your backend endpoint.
3. **Run**: `npm run dev` to launch the high-fidelity interface locally.
